ECS 10                                                             Program #5 (50 points)                                                Summer 2009

 

Due: Tuesday, July 28th, 11:59pm.  Filename: airline.py.  No flow charts, error journals, or written homework.

New concepts:  sort, search, list of lists, menus, and exception handling.

 

            You are to write an airline reservation program for a small shuttle airline called Fly By Night Airlines.   The airline will have only four flights between three cities.  The airline allows reservations for specific seats on the plane.

When a customer reserves seat(s) on a flight, the program asks for his/her name.   The program will then associate the name with a seat reservation.   A seat reservation contains a flight number and the number of seats reserved by the customer.   When a customer wants to cancel a flight, the program asks for 1) flight number, 2) seats to cancel, and 3) the customer's name.   The program will only cancel the customer’s reservation if he/she has already reserved that number (exactly) of seats for the flight.  The management also wants to be able to see an alphabetically sorted list of all customers that have reserved seats.  The list should be sorted by last names. 

 

Specifications:

1.      Flights

1.1.   Each of the flights has associated with it a flight number, an origin city, a destination city, and the number of seats reserved.   Each flight can only hold six passengers. 

2.      Files: There are two files associated with the reservation system.

2.1.   Both files are available from the course website.  Copy these files to the directory in which you write your program.  To make debugging easier, neither file should be updated when reservations are changed.

2.2.   The values for all of this information for all of the flights are stored in the file flights.txt.  The format for this file is:

Description:       Flight #  comma   Origin   comma Reserved comma Destination

Data type:         <int>                 <string>              <int>                   <string>

2.3.    passenger.txt contains a list of seat reservations with the associated passenger names.  The format of each line in passengers.txt is:

Description: FlightNo  comma  NumSeats comma  Name

Data type:       <int>                     <int>                <string>

3.      Menu system

3.1.   Implement and provide a menu for the following choices:  1) Make a reservation;  2) Cancel a reservation; and  3)  Display all information about all flights.

3.2.   To make a reservation, the program 1) displays flight numbers with their cities;  2) reads flight number 3) reads number of seats to reserve; 4) notifies the  user whether the request can be satisfied;  5) if the request is an acceptable then update the number of seats reserved for the plane.

3.3.   To cancel a reservation, the program 1) displays flight numbers with their cities;  2) reads flight number 3) reads number of seats to cancel; 4) notifies the user whether the request can be satisfied (to satisfy such a request there must be at least as many seats already reserved as the cancellation requests to cancel);  5) if the request is an acceptable then update the number of seats reserved for the plane.

3.4.   The flight information display should have the following format:

3.4.1. Flight # (3 digits) <space> Origin(10 characters) <space> Destination(10 characters) <space>  Reserved(1 digit)

4.      Implementation: 

4.1.   Origin city, destination city, and reserved seats of the flights must all be stored in a list of lists. 

4.2.   Each passenger name will only be associated with one flight reservation.  That means you don’t have to worrry about one customer having two reservations.  (This makes the name search easier, since you may stop the search when you find the first name match.)    The information about seat reservations must be maintained in a list of lists that holds the flight number, seats reserved, the first name, and last name of each passenger.

5.      Testing:  A testing regimen of valid input data is supplied on the back.   Your program must be robust!  You may not assume that the user will enter integers where appropriate. 

6.      Hints:  Remember to use top-down design to break the program into manageable procedures.  Notice that the processes for making and canceling reservations are almost identical.  Once you have one those functions written and debugged, just copy it and change a few words and conditions to create the other function.  These functions should call at least three other functions.
